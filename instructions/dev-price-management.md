# 0. 대원칙

- 인프라 비용은 0원에 수렴해야 함
- 무료 플랜 범위 내 운영
- 서버리스 기반 유지
- 외부 유료 CDN 금지
- 캐시 전략을 적극 활용하여 비용 최소화

---

# 1. 캐시 전략 (Cost Optimization Core)

## 1.1 정적 페이지 캐시

- 블로그 글은 SSG 또는 ISR 기반으로 생성
- Vercel Edge Cache 활용
- 게시글 수정 시에만 재생성

목표:
- DB 조회 최소화
- 트래픽 증가 시에도 서버 호출 최소화

---

## 1.2 이미지 캐시 전략

- Supabase Storage 사용
- Cache-Control 헤더 설정 (long max-age)
- 가능하면 immutable 파일명 전략 사용
  - 예: image_hash.png

목표:
- 동일 이미지 반복 다운로드 방지
- 브라우저 캐시 적극 활용

---

## 1.3 API 캐시

- 카테고리 트리 → 정적 캐시
- 최근 글 목록 → ISR 또는 revalidate 전략
- 댓글은 실시간성 필요 시 최소 TTL 캐시

---

## 1.4 LLM 응답 캐시 (매우 중요)

- 동일 질문에 대한 응답 DB에 저장
- 질문 텍스트를 해시하여 캐시 키 생성
- 동일 질문 재요청 시 OpenAI 호출 생략

목표:
- LLM 호출 비용 최소화
- 반복 질문 무료 처리

---

## 1.5 RAG 문서 캐시

- 게시글 임베딩은 생성 후 영구 저장
- 재임베딩 금지 (수정 시에만 재생성)

---

# 2. 블로그 기반 질의응답 비용 제어

- 상위 N개 문서만 사용 (3~5개 제한)
- 토큰 수 제한
- 대화 히스토리 최소화 또는 미저장
- 스트리밍 응답 시 추가 호출 금지

---

# 3. 이미지 정책

- 업로드 시 자동 압축 권장
- 대용량 원본 업로드 금지
- 썸네일 별도 저장 (선택)

---

# 4. 트래픽 대응 전략

트래픽 증가 시에도:

1. 정적 페이지 캐시가 1차 방어
2. 이미지 브라우저 캐시가 2차 방어
3. LLM 응답 캐시가 3차 방어

OpenAI 호출은 항상 최후의 수단

# 3. 사용자 유형

# 4. 기능 요구사항

## 4.1 블로그 CRUD

- 게시글 생성(Create)
- 게시글 조회(Read)
- 게시글 수정(Update)
- 게시글 삭제(Delete)

### 제약

- 관리자만 생성/수정/삭제 가능
- 공개 블로그 (비로그인 조회)

---

## 4.2 카테고리 시스템

### 요구사항

- 정확히 3레벨 계층 구조
- 게시글은 반드시 3레벨 카테고리에 속해야 함

예시:

```
1레벨: Tech
  └ 2레벨: AI
       └ 3레벨: RAG
```

### 데이터 모델

- self-referencing category 구조
- level 컬럼 명시 관리

---

## 4.3 댓글 기능

- 방문자 댓글 작성 가능
- 게시글 단위 댓글
- 관리자 댓글 삭제 가능
- 대댓글 여부 → ❓결정 필요

---

## 4.4 편집기 지원

- 이미지 삽입 지원
- 링크 지원

### 이미지 처리

- Supabase Storage 업로드
- 외부 이미지 자동 리사이징 서비스 사용 금지
- 빌드 타임 최적화 또는 클라이언트 리사이징 고려
@dev-price.management.md 참고

---

## 4.5 블로그 기반 질의응답 기능 (RAG)

### 요구사항

- OpenAI API 사용
- 블로그 글 기반 응답
- 외부 지식 최소화

### 동작 정의

#### 1️⃣ 글 작성 시

- 텍스트 분할
- 임베딩 생성
- pgvector 저장

#### 2️⃣ 질문 입력 시

- 질문 임베딩 생성
- 유사 문서 상위 N개 검색 (비용 최소화 위해 3~5개 제한)
- 검색 문서 context로 LLM 호출
- 응답 반환

### 비용 제어 전략

- 토큰 제한 설정
- temperature 고정
- 대화 히스토리 최소화 또는 미저장 전략

# 5. UI 요구사항

## 5.1 레이아웃

### 좌측 영역

- 3레벨 카테고리 트리 패널
- 클릭 시 필터링

---

## 5.2 메인 영역

### 상단

- LLM 질문 입력 박스

placeholder 예:

- “이 블로그에서 말하는 연애관은?”
- “작성자가 생각하는 AI의 미래는?”

---

### 기본 상태

- 최근 글 순서대로 리스트 표시
- 정렬 기준: created_at DESC
- 커서 기반 무한 스크롤

각 항목:

- 제목
- 요약
- 카테고리
- 작성일
- 썸네일 이미지 (선택)

---

### 질문 입력 시

- 글 리스트 영역 숨김
- 채팅 UI 전환
- 블로그 기반 챗 시작

---

### 채팅 UI

- 사용자 질문 표시
- AI 답변 표시
- 대화 히스토리 저장 여부 → 저장하지 않음
- 새로고침 시 초기화 여부 → 초기화

---
